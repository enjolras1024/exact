Bindings

Exact中的绑定是基于事件而实现的。这意味着，即便不使用绑定，你也能完成数据的传递。 当然，使用绑定自有好处，它以简单直观的方式极大地简化了繁琐重复的赋值操作过程。 

<input type="text" value@="$.message" readonly x-id="$msg">

几乎等价于：

this.on('changed.message', function() {
  this.$msg.set('value', this.message);
});

Key Terms

在Exact中使用好绑定，需要了解几个关键词。

（1）绑定模式：基本上有3种绑定模式可选，即单次、单向和双向。其中单次绑定是单向绑定中特殊的一种绑定，它只在绑定建立时立即起效，不关心之后的变化。

（2）绑定路径：绑定表达式中出现的属性链即为绑定路径。路径的源头通常是组件本身，但也可以是已知的某个局部变量对象。

（3）绑定对象和绑定对象属性：绑定的目的是要将绑定表达式的运算结果赋给一个对象的属性。 对于单次或单向绑定，这样的对象是任意的，这样的对象属性也是任意的。 对于双向绑定，则要求绑定对象亦可作为绑定源，绑定对象属性亦可作为绑定源属性。

（4）绑定源和绑定源属性：绑定源是绑定路径上数据的直接提供者。绑定源属性的值改变后通常会派发一个属性更改事件，但有时普通属性也可作为绑定源属性，参见（5）。

（5）绑定域和绑定域事件：绑定表达式依托的上下文就是绑定域，通常这样的上下文就是一个组件。没有特别声明的情况下，一个绑定会关心每个绑定路径对应的属性更改事件。 但是，你也可以指定一个绑定域上的事件作为绑定域事件。这样，绑定就只关心指定绑定域事件有没有发生，而无需关心多个属性更改事件。

（6）求值器和转换器：每个绑定都有一个求值器，负责根据一些常量和绑定域上一个或多个绑定路径来求出结果值。 转换器是可选和多选的，负责将求值器求得的值转换成其他形式。

Basic Syntax

模板中绑定的语法是由Exact的HTMX解析器规定的，文本插值绑定的基本语法如下：

<!--<绑定模式符>{<求值器表达式>[|转换器表达式] [*绑定域事件]}-->
<!--单次绑定-->
<p>Tip: &{ $.tip }</p>
<!--单向绑定，无参转换器-->
<h1>@{ $.title | upper }</h1>
<!--单向绑定，带常量参数的转换器-->
<span>@{ $.time | format('hh:mm:ss') }</span>
<!--单向绑定，带变量参数的转换器-->
<span>@{ $.text | slice(0, $.length) }...</span>

元素或组件属性绑定的语法，看起来就像是运算符重载：

<!--<绑定对象属性名><绑定模式符>="<求值器表达式>[|转换器表达式] [*绑定域事件]"-->
<!--单次绑定-->
<p style&="$.styleString"></p>
<!--单向绑定，无参转换器-->
<h1 title@="$.title | upper"></h1>
<!--单向绑定，带参转换器-->
<input type="text" value@="$.time | format('hh:mm:ss')">
<!--双向绑定，双向转换器-->
<input type="text" x-type="TextBox" value#="$.color | hex2rgb">

对于双向绑定，特别要求求值器表达式为单个路径，以保持对称关系。 若在双向绑定使用转换器，转换器也必须是双向的。关于求值器和转换器更多的使用细节可参见下文。

如果要将文本插值的结果赋给元素或组件的某个属性，则可以这样书写：

<a href?="http://www.xxx.com/page/@{$.page}/"
   style?="color: black; font-size: @{ $.fontSize }px;"
   class?="btn-next @{$.page >= $.total ? 'disabled' : ''}">
  下一页
</a>

特别地，对于class和style的绑定，Exact提供一种风格一致的扩展语法：

<!--<绑定对象属性名><绑定模式符>:<求值器表达式>[|转换器表达式] [*绑定域事件];-->
<a x-class="btn-next: true; disabled@: $.page >= $.total;"
   x-style="color: black; fontSize?: @{ $.fontSize }px;">
</a>

即将class和style视作对象而非字符串，然后独立地为其中某个属性添加绑定。

Optional Converters

一些场景下，在视图上呈现数据时需要有一个格式化的过程，比如标题要大写：

{
  descriptors: ['title'],
  resources: {
    format: function(value, brackets) {
     return brackets[0] + value.toUpperCase() + brackets[1];
    }
  },
  $template: '<h1>@{$.title | format(\'##\')}</h1>'
}
// this.title = 'hello' => <h1>#HELLO#</h1>

等价地，

var format = function(value, brackets) {
   return brackets[0] + value.toUpperCase() + brackets[1];
}
var args = ['title'];
args.flags = [1]; // flag： 0=>常量（默认），1=>路径，2=>求值器
​
Exact.Binding.build(this.header, 'title', this, {
  mode: 1,
  scopePaths: ['title'],
  evaluator: Exact.Evaluator.makeGetEvaluator(args),
  converters: [Exact.Evaluator.makeEvaluator(format, ["##"])]
})

可见转换器本质上也是求值器，只是它接受的第一个参数为前面已求得的值。 可以在全局RES、本地resources或绑定域上注册转换器，可以使用多个转换器（建议不超过2个），在声明式绑定表达式中以符号'|'连接。 更多示例如下：

// 本地resources或全局RES中定义了函数（或转换器对象）upper和wrap；
@{$.title | upper | wrap('##')}
​
// 转换器中可以使用路径作为变量，当title或upperOrNot改变时都有可能触发绑定
@{$.title | $.format($.upperOrNot, '##')}
// 等价于
@{$.format($.title, $.upperOrNot, '##')}
​
// 全局RES上注册双向转换器
Exact.RES.register('num2str', {
  exec: String, //正向
  back: Number  //反向
};
<input type="text" x-type="TextBox" value#="$.price | num2str">

JavaScript Expressions

声明式定义绑定表达式时，除了直接书写单个的绑定路径外，你还可以在其中书写简单的JavaScript表达式，例如：

@{ $.state === 'ok' }
@{ $.firstName + ' ' + $.lastName }
@{ $.amount > 100 ? 'large' : 'normal' }

对于上述含有运算符的表达式，Exact使用new Function(...)将JavaScript表达式转化成函数， 存在一定的限制和问题，例如无法访问本地resources和全局RES中的内容。 我们建议在声明式定义的绑定表达式中尽量使用函数形式的表达式，例如：

{
  resources: {
    equal: function(a, b) { return a === b; }
  },
  $template: '<a x-class="active@: equal($.state, \'ok\');"></a>'
}

Using Event in Expression

默认地，当监听到绑定表达式中某个属性发生变化后，该绑定会自动重新求值并赋值。 但你也可以指定一个事件，只当该事件发生时，该绑定才会重新求值并赋值。

<div x-type="Comp" total@="$.a + $.b + $.c + $.d *changed.d"></div>

如果省略事件类型名，默认会监听`updated`事件。

<div x-type="Comp" total@="$.a + $.b + $.c + $.d *"></div>

Using Depends in Descriptors

虽然Exact支持的绑定表达式是丰富的，但是Exact建议不要书写冗长而难以阅读的绑定表达式。 如果一个绑定表达式含有多个绑定路径或者逻辑过于复杂，你可以考虑在`descriptors`中定义一个依赖于其他绑定路径的属性，像这样：

{
  $template: '<h2>@{ $.fullName }</h2>'
  descriptors: [{
    fullName: {
      depends: ['firstName', 'lastName'],
      get: function(props) {
        return this.firstName + ' ' + this.lastName;
      }
      //,set: function(value, props) {
      //    var names = value.split(' ');
      //  this.save({firtName: names[0], lastName: names[1]});
      //}
    }
  }]
}
