Elements + Components

Exact elements are shadows of DOM elements. A component is a shadow of DOM tree, composed of texts, elements and custom components. When you update the shadow tree, Exact helps you render the dirty parts of a DOM tree in batch mode.

Declaration | Procedure

Usually you can declaratively define a rich template (including expressions) for a component class. The once compiled template will be used for initialization. Or sometimes you just need a pure template and add children, event listeners in later procudure (after initialization).

Events & Bindings

Events make the UIs live and controllable. One-way, two-way bindings and template strings are used for passing data in an intuitive way. And you can detect each property-changed event without using bindings, since they are base on custom events.
